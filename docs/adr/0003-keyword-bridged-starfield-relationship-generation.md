# Keyword-Bridged Starfield Relationship Generation

The Starfield generation flow will derive candidate Passage Relationships by first asking AI-agent to produce Passage Keywords for each Passage, then merging highly similar keywords into Canonical Passage Keywords, and finally proposing Keyword-Derived Relationships between Passages that share a canonical keyword. Direct keyword matches create `same_topic` candidates only; richer Relationship Types such as `prerequisite`, `further_reading`, `problem_solution`, and `comparison` require a second AI-agent pass over the connected Passage Text. This keeps Passage as the only graph node type, avoids an expensive and noisy all-pairs Passage comparison, and preserves a clear evidence trail for why an edge exists without turning keywords into visible concept nodes.
